# State

## 今日の目的

Stateが、画面の現在の状態を表すデータであることを理解する。

## 作ったもの

ボタンを押すと、カウント数のStateが変わり、画面の数字も変わるページ。

## アプリケーションURL

http://localhost:5500/src/state-practice/index.html

## 重要な用語

| 用語 | 意味 | 自分の言葉で言うと |
|---|---|---|
| State | 画面の現在の状態を表すデータ | 今の画面がどういう状態かを覚えておく値 |
| `let` | あとから値を変更できる変数 | 変わるデータに名前をつける書き方 |
| `count` | 今回のカウント数を表すState | 今の数字を覚えておくデータ |
| `showCount` | Stateを画面に表示する関数 | 今のStateを画面に反映する処理 |

## Stateの説明

Stateは、画面の「今の状態」を表すデータ。

画面に表示されている内容そのものではなく、表示のもとになるデータとして考える。

今回の例では、画面に表示される数字のもとになっている `count` がState。

```js
let count = 0;
```

ボタンを押すと `count` が変わる。`count` が変わっただけでは画面は自動では変わらないため、`showCount()` でDOMを通じて画面に反映している。

```text
Stateが変わる
↓
画面表示を更新する処理を呼ぶ
↓
DOMを通じてHTMLを書き換える
↓
画面が変わる
```

## Stateの例

| アプリの例 | Stateの例 | 画面への影響 |
|---|---|---|
| カウントアプリ | 現在のカウント数 | 表示される数字が変わる |
| ToDoアプリ | タスク一覧、入力中の文字、完了状態 | 一覧やチェック状態が変わる |
| クイズアプリ | 現在の問題番号、選んだ答え、正解かどうか | 問題文や結果表示が変わる |
| 検索アプリ | 検索文字、表示中のデータ | 表示される一覧が絞り込まれる |

## 処理の流れ

```text
ユーザーが「1増やす」ボタンを押す
↓
clickイベントが発生する
↓
increaseCount関数が実行される
↓
count = count + 1 でStateが変わる
↓
showCount関数が実行される
↓
countText.textContentで画面の数字が更新される
```

## Stateと画面表示の対応

| State | 画面上の表示 | 変わるきっかけ |
|---|---|---|
| `count` | `現在のカウント` の数字 | 「1増やす」ボタンを押したとき |

## コードと画面の対応

| 画面上の要素 | コード上の場所 | 役割 |
|---|---|---|
| カウント表示 | `p id="countText"` | Stateの値を表示する |
| 1増やすボタン | `button id="increaseButton"` | Stateを変更する操作を受け取る |
| State | `let count = 0` | 現在のカウント数を覚えておく |
| State変更 | `count = count + 1` | カウント数を1増やす |
| 画面更新 | `countText.textContent = count` | Stateの値を画面に表示する |

## 動作確認方法

1. ブラウザで http://localhost:5500/src/state-practice/index.html を開く。
2. 最初に「現在のカウント」が `0` と表示されるか確認する。
3. 「1増やす」ボタンを押すたびに、数字が1ずつ増えるか確認する。

### 1. Stateとは何ですか？

Stateは、現在の画面の状況を表すもの。

### 2. 今回のページでは、どの変数がStateですか？

今回のページでは、カウント数を表す `count` という変数がState。

### 3. ボタンを押してから数字が変わるまで、どんな順番で処理が進みますか？

ボタンを押す。

↓

イベントが起きる。

↓

JavaScriptの処理が動く。

↓

`count` というStateが変わる。

↓

JavaScriptがDOMを通じてHTMLを書き換える。

↓

画面の数字が変わる。
