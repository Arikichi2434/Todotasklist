# データの流れ

## 今日の目的

データがどこから来て、どの処理を通って、どう画面に表示されるかを理解する。

## 作ったもの

JavaScriptの配列データを、画面のリストとして表示するページ。

## アプリケーションURL

http://localhost:5500/src/data-flow-practice/index.html

## 重要な用語

| 用語 | 意味 | 自分の言葉で言うと |
|---|---|---|
| データ | 画面に表示したり処理したりする値 | アプリが扱う中身 |
| 配列 | 複数のデータをまとめて持つ形 | データのリスト |
| 元データ | 加工前のデータ | 最初に持っているデータ |
| 表示データ | 画面に表示される形になったデータ | ユーザーに見える形 |
| `forEach` | 配列の中身を1つずつ処理する命令 | リストの項目を順番に取り出す |
| `innerHTML` | HTML要素の中身をHTML文字列で変更する命令 | リストの中身を作り直す |

## データフローの図

```text
元データ
tasks配列
↓
処理
showTasks関数
↓
繰り返し
tasks.forEach
↓
HTML化
"<li>" + task + "</li>"
↓
DOM更新
taskList.innerHTML
↓
画面表示
タスク一覧のリスト
```

## 入力・処理・出力で見るデータフロー

| 区分 | 今回のコード | 役割 |
|---|---|---|
| 入力 | `tasks` 配列 | 表示したい元データ |
| 処理 | `showTasks()` | データを画面表示用に変換する |
| 加工 | `tasks.forEach(...)` | 配列の中身を1つずつ取り出す |
| 出力 | `taskList.innerHTML` | 画面にリストとして表示する |

## データフローで大事な考え方

元データと画面表示は同じものではない。

元データはJavaScriptの中にある。画面表示は、JavaScriptが元データをHTMLの形に変換し、DOMに反映した結果として見えている。

## 処理の流れ

```text
JavaScriptにtasks配列がある
↓
showTasks関数が実行される
↓
taskList.innerHTML = "" で表示エリアを空にする
↓
tasks.forEachでデータを1つずつ取り出す
↓
取り出した文字をliタグにする
↓
taskList.innerHTMLに追加する
↓
画面にリストとして表示される
```

## 元データ・処理・表示結果

| 種類 | コード上の場所 | 内容 |
|---|---|---|
| 元データ | `tasks` | 学習タスクの文字列が入った配列 |
| 加工処理 | `tasks.forEach(...)` | 配列から1つずつ文字列を取り出す |
| HTML化 | `"<li>" + task + "</li>"` | 文字列をliタグにする |
| 表示結果 | `ul id="taskList"` | 画面上のリスト |

## コードと画面の対応

| 画面上の要素 | コード上の場所 | 役割 |
|---|---|---|
| タスク一覧 | `ul id="taskList"` | 配列データを表示する場所 |
| 元データ | `const tasks = [...]` | 表示するタスクをまとめて持つ |
| 一覧表示処理 | `showTasks()` | 配列データを画面に表示する |
| 繰り返し処理 | `tasks.forEach(...)` | タスクを1つずつ取り出す |
| 画面更新 | `taskList.innerHTML = ...` | HTMLとしてリストを表示する |

## 動作確認方法

1. ブラウザで http://localhost:5500/src/data-flow-practice/index.html を開く。
2. 3つの学習タスクがリストとして表示されるか確認する。
3. `script.js` の `tasks` 配列と、画面上のリストが対応しているか確認する。

## 自分の言葉で説明する練習

### 1. 今回の元データは、どのコードですか？

今回の元データは、`script.js` の `const tasks = [...]`。

```js
const tasks = [
  "HTMLの役割を説明する",
  "CSSの役割を説明する",
  "JavaScriptの役割を説明する"
];
```

### 2. `forEach` は何をしていますか？

`forEach` は、配列の中にあるデータを1つずつ取り出して、指定した処理を実行している。

注意点: `forEach` 自体がHTMLを返しているわけではない。今回のコードでは、`forEach` の中の処理で `"<li>" + task + "</li>"` というHTML文字列を作っている。

### 3. データが画面に表示されるまで、どんな順番で処理が進みますか？

JavaScriptに `tasks` 配列がある。

↓

`showTasks` 関数が動く。

↓

`forEach` で配列のデータを1つずつ取り出す。

↓

取り出したデータを `li` タグのHTML文字列に変換する。

↓

`taskList.innerHTML` にHTML文字列を入れる。

↓

ブラウザがHTMLとして表示する。

↓

CSSのルールに合わせて見た目が整う。
