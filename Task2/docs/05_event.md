# イベント

## 今日の目的

イベントが、ユーザー操作をきっかけに発生する合図であることを理解する。

## 作ったもの

入力欄に文字を入れると、入力中の文字が画面に表示されるページ。

## アプリケーションURL

http://localhost:5500/src/event-practice/index.html

## 重要な用語

| 用語 | 意味 | 自分の言葉で言うと |
|---|---|---|
| イベント | ユーザー操作などをきっかけに発生する合図 | クリックや入力が起きた知らせ |
| `input` イベント | 入力欄の内容が変わったときに発生するイベント | 文字を入力するたびに起きる合図 |
| `addEventListener` | イベントが起きたときに動かす処理を登録する命令 | 何が起きたら何をするか決める |
| `value` | 入力欄に入っている現在の値 | 入力欄の中の文字 |

## 主要なイベント

| イベント名 | いつ発生するか | よく使う場面 |
|---|---|---|
| `click` | ボタンや要素がクリックされたとき | ボタンを押して処理を実行する |
| `input` | 入力欄の内容が変わったとき | 入力中の文字をすぐ画面に反映する |
| `change` | 入力や選択の内容が確定して変わったとき | セレクトボックスやチェックボックスの変更を扱う |
| `submit` | フォームが送信されたとき | 入力フォームの送信処理を行う |
| `keydown` | キーボードのキーが押されたとき | Enterキーなどのキー操作に反応する |

## イベントの基本形

```js
要素.addEventListener("イベント名", 実行する関数);
```

例:

```js
memoInput.addEventListener("input", showInputText);
```

このコードは、「`memoInput` の入力内容が変わったら、`showInputText` 関数を実行する」という意味。

## 処理の流れ

```text
ユーザーが入力欄に文字を入れる
↓
inputイベントが発生する
↓
showInputText関数が実行される
↓
memoInput.valueで入力中の文字を取得する
↓
previewText.textContentで画面の文字を書き換える
```

## コードと画面の対応

| 画面上の要素 | コード上の場所 | 役割 |
|---|---|---|
| 入力欄 | `input id="memoInput"` | ユーザーが文字を入力する |
| 表示エリア | `p id="previewText"` | 入力中の文字を表示する |
| 入力欄の取得 | `document.querySelector("#memoInput")` | JavaScriptから入力欄を取得する |
| 表示エリアの取得 | `document.querySelector("#previewText")` | JavaScriptから表示エリアを取得する |
| イベント登録 | `memoInput.addEventListener("input", showInputText)` | 入力されたときに関数を実行する |
| 入力値の取得 | `memoInput.value` | 入力欄の現在の文字を読む |
| 画面更新 | `previewText.textContent = memoInput.value` | 入力中の文字を画面に表示する |

## 動作確認方法

1. ブラウザで http://localhost:5500/src/event-practice/index.html を開く。
2. 入力欄に文字を入力する。
3. 入力した文字が「入力中の文字」の下に表示されるか確認する。

## 自分の言葉で説明

* イベントは、ユーザー操作をきっかけに発生する合図。
* `input` イベントは、入力欄の文字が変わるたびに発生する。
* `addEventListener` は、イベントが起きたときに動かす処理を登録する。

## 自分の言葉で説明する練習

### 1. イベントとは何ですか？

イベントとは、ブラウザ上でユーザーが行う操作をきっかけに発生する合図。

注意点: イベントは「操作そのもの」ではなく、「クリックされた」「入力された」とブラウザが知らせる合図として考える。

### 2. `input` イベントは、いつ発生しますか？

`input` イベントは、ユーザーが入力欄に文字を打ち込み、入力欄の内容が変わったときに発生する。

### 3. 入力欄に文字を入れてから画面に表示されるまで、どんな順番で処理が進みますか？

ユーザーが入力欄に文字を入れる。

↓

入力欄の内容が変わる。

↓

`input` イベントが発生する。

↓

JavaScriptの `showInputText` 関数が動く。

↓

`memoInput.value` で入力欄の文字を読み取る。

↓

`previewText.textContent` でHTMLの表示を書き換える。

↓

画面に入力した文字が表示される。
