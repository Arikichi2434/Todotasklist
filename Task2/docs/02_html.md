# HTML

## 今日の目的

HTMLが画面の構造を作る役割を持っていることを理解する。

## 作ったもの

見出し、文章、入力欄、ボタン、リストを持つHTMLページ。

## アプリケーションURL

http://localhost:5500/src/html-practice/index.html

## 重要な用語

| 用語 | 意味 | 自分の言葉で言うと |
|---|---|---|
| HTML | 画面の構造を作る言語 | 画面に何を置くかを決めるもの |
| タグ | HTMLの部品を表す書き方 | 見出しや文章などの種類を表す印 |
| h1 | ページの大きな見出し | タイトルのようなもの |
| p | 文章 | 説明文を書く場所 |
| input | 入力欄 | ユーザーが文字を入れる場所 |
| button | ボタン | ユーザーが押す部品 |
| ul / li | リスト | 複数の項目を並べる部品 |

## HTML用語集

| コード・用語 | 役割 | 例 |
|---|---|---|
| `<!DOCTYPE html>` | HTML文書であることをブラウザに伝える | `<!DOCTYPE html>` |
| `<html>` | HTML全体を囲む | `<html lang="ja">` |
| `lang="ja"` | ページの言語が日本語であることを表す | `<html lang="ja">` |
| `<head>` | 画面には直接出ない設定を書く場所 | `<head>...</head>` |
| `<meta charset="UTF-8">` | 文字コードを指定する | `<meta charset="UTF-8">` |
| `<meta name="viewport">` | スマホなどの表示幅に対応するための設定 | `<meta name="viewport" content="width=device-width, initial-scale=1.0">` |
| `<title>` | ブラウザのタブに表示されるタイトル | `<title>HTML Practice</title>` |
| `<body>` | ブラウザ画面に表示される中身を書く場所 | `<body>...</body>` |
| `<main>` | ページの中心となる内容を表す | `<main>...</main>` |
| `<section>` | 内容のまとまりを表す | `<section>...</section>` |
| `<h1>` | ページで一番大きな見出し | `<h1>フロントエンド学習メモ</h1>` |
| `<h2>` | 小さめの見出し | `<h2>今日の学習テーマ</h2>` |
| `<p>` | 文章を表す | `<p>HTMLは構造を作ります。</p>` |
| `<input>` | ユーザーが入力する欄 | `<input type="text">` |
| `placeholder` | 入力前に表示される案内文 | `<input placeholder="学んだことを書く">` |
| `<button>` | ユーザーが押せるボタン | `<button>追加する</button>` |
| `<ul>` | 順番のないリスト全体 | `<ul>...</ul>` |
| `<li>` | リストの1項目 | `<li>p: 文章</li>` |
| `id` | 1つの要素を特定するための名前 | `<p id="message">...</p>` |
| `class` | 複数の要素に使える分類名 | `<main class="page">...</main>` |

## 処理の流れ

```text
HTMLを書く
↓
ブラウザがHTMLを読み込む
↓
タグの意味に合わせて画面の部品を表示する
```

## コードと画面の対応

| 画面上の要素 | コード上の場所 | 役割 |
|---|---|---|
| 大きな見出し | `h1` | ページのタイトルを表示する |
| 説明文 | `p` | 文章を表示する |
| 入力欄 | `input` | ユーザーが文字を入力する |
| ボタン | `button` | ユーザーが押せる部品を作る |
| リスト | `ul` と `li` | 複数の項目を一覧で表示する |

## 動作確認方法

1. ブラウザで http://localhost:5500/src/html-practice/index.html を開く。
2. 「フロントエンド学習メモ」という見出しが表示されるか確認する。
3. 入力欄、ボタン、リストが表示されるか確認する。

## 自分の言葉で説明

* HTMLは、画面に必要な部品と構造を作るもの。
* タグは、画面上の部品の種類を表すもの。
* HTMLだけでは、見た目の細かい調整や動きはまだ作れない。

## 自分の言葉で説明する練習

### 1. HTMLはWebページの中で何を担当していますか？

Webページの骨組み、つまり部品構成を担当している。

### 2. `h1`、`p`、`input`、`button` はそれぞれ何を表していますか？

* `h1` は、一番重要な見出しを表す。
* `p` は、文章を表す。
* `input` は、ユーザーが文字などを入力する入力欄を表す。
* `button` は、ユーザーが押せるボタンを表す。

### 3. 今回のページにCSSやJavaScriptがまだない場合、何ができて何ができませんか？

HTMLだけでも、見出し、文章、入力欄、ボタン、リストなどの部品は表示できる。

ただし、CSSがないので見た目の整理はできない。JavaScriptがないので、ボタンが押されたときの処理もまだできない。
