# Webアプリ全体像

## 今日の目的

Webアプリの中で、フロントエンド・バックエンド・DBがそれぞれ何を担当しているかを理解する。

## 作ったもの

Webアプリ全体の役割と流れをまとめた学習メモ。

## 重要な用語

| 用語 | 意味 | 自分の言葉で言うと |
|---|---|---|
| フロントエンド | ユーザーが直接見る画面や操作する部分 | 画面を作って、ユーザーの操作を受け取る場所 |
| バックエンド | データ処理や保存の判断をする裏側の部分 | 画面から見えないところで処理する場所 |
| DB | データを保存しておく場所 | アプリのデータをしまっておく箱 |
| ブラウザ | Webページを表示するアプリ | HTML、CSS、JavaScriptを読み取って画面にするもの |
| API | フロントエンドとバックエンドがやり取りする入口 | 画面側とサーバー側をつなぐ窓口 |

## Webアプリの構成図

```text
ユーザー
↓
ブラウザ
↓
フロントエンド
↓
API
↓
バックエンド
↓
DB
```

## 画面が表示されるまでの流れ

```text
ユーザーがWebアプリを開く
↓
ブラウザがHTML、CSS、JavaScriptを読み込む
↓
HTMLで画面の構造を作る
↓
CSSで見た目を整える
↓
JavaScriptで動きを準備する
↓
画面が表示される
```

## ユーザー操作後の流れ

```text
ユーザーが画面を操作する
↓
イベントが発生する
↓
JavaScriptの処理が動く
↓
必要に応じてAPIに通信する
↓
バックエンドが処理する
↓
必要に応じてDBからデータを取り出す
↓
結果がフロントエンドに返る
↓
画面が更新される
```

## フロントエンドとバックエンドの違い

| 項目 | フロントエンド | バックエンド |
|---|---|---|
| 役割 | 画面を表示し、操作を受け取る | データ処理や保存の判断をする |
| 主な場所 | ブラウザ | サーバー |
| 使うもの | HTML、CSS、JavaScript | サーバー側の言語、API、DB |
| ユーザーから見えるか | 見える | 基本的には見えない |

## DBが必要な理由

DBは、ユーザー情報やタスク一覧などのデータを、あとからまた使えるように保存しておくために必要。

## コードと画面の対応

| 画面上の要素 | コード上の場所 | 役割 |
|---|---|---|
| まだ画面は作っていない | なし | 今回は全体像の整理だけを行う |
